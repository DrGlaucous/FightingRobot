For figuring out how to parse dshot telemetry


to find the value we need:
timing values for each dshot speed (E.G 25 us)
multiply the timing value by 90% (Value 'A')
divide each telemetry symbol by 'A' to get the number of bits each one carries
if we come up short, replace any trailing bits with '1' (due to the way RMT readback stops)
remove CRC and calculate the checksum, ignore any bad packets



for dshot 300, each bit is 25 us, +-10%
